# app.py 개선 계획 (완료)

## 배경

`poc.py`는 "JSON 파일로 데이터를 읽고 쓸 수 있는가"만 빠르게 확인해본 vibe-coding PoC이고, `app.py`는 그 구조를 그대로 살려 만든 실제 CRUD 콘솔 앱이다. PoC 단계와 달리 이제부터는 실제로 계속 쓰일 코드이므로, 재검토하며 아래 문제를 찾아 개선한다. **`poc.py`는 PoC 당시 코드를 그대로 남겨두는 참고 자료이므로 수정하지 않는다.**

## 발견한 이슈와 개선 방향

1. **저장이 원자적이지 않음** — `save()`가 `data.json`을 바로 열어 덮어쓰기 때문에, 쓰는 도중 프로그램이 죽거나 디스크 오류가 나면 파일이 반쯤 쓰인 상태로 손상될 수 있다. → 임시 파일에 먼저 쓰고 `os.replace()`로 원자적으로 교체하도록 변경.
2. **손상된 JSON 파일 처리 없음** — `data.json`이 어떤 이유로든 깨진 상태라면 `load()`가 `json.JSONDecodeError`를 그대로 던져 프로그램이 죽는다. → 예외를 잡아 경고 메시지를 출력하고 빈 목록으로 시작하도록 변경.
3. **ID 입력 검증 로직 중복** — `update()`와 `delete()`가 "ID를 입력받아 정수 변환하고, 실패하면 에러 출력 후 종료"하는 동일한 코드를 각각 갖고 있다. → `_prompt_item_id()` 헬퍼로 추출.
4. **빈 이름 생성 허용** — `create()`가 빈 문자열 이름도 그대로 저장한다. → 빈 이름이면 생성을 거부하고 에러 메시지 출력.
5. **테스트 커버리지 공백** — `read_all()`, `read_one()`은 테스트가 전혀 없었다. → 각각 정상 조회/빈 목록/검색 실패 케이스 테스트 추가. 손상된 JSON 처리, 빈 이름 거부에 대한 회귀 테스트도 추가.
