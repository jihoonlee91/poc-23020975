# poc-23020975

JSON 파일을 데이터 저장소로 사용하는 CRUD 콘솔 애플리케이션 프로젝트입니다.
"이 방식이 실제로 동작하는가?"를 빠르게 확인해본 PoC(Proof of Concept)와,
그 구조를 그대로 살려서 만든 실제 애플리케이션 두 가지로 구성되어 있습니다.

## 구성 파일

| 파일 | 설명 |
| --- | --- |
| `poc.py` | **PoC** — JSON 파일을 읽고(`load`) 쓰는(`save`) 최소 기능만 빠르게 검증한 프로토타입. 직접 실행하면 add/update/delete 예시가 순서대로 출력됩니다. |
| `app.py` | **실제 앱** — `poc.py`의 코드 구조(`load`/`save`/`find`)를 그대로 활용해 만든 대화형 CRUD 콘솔 프로그램입니다. |
| `tests/test_app.py` | `app.py`의 각 기능(생성/조회/수정/삭제, 잘못된 입력 처리 등)을 검증하는 pytest 테스트입니다. |
| `data.json` | 실행 시 자동으로 생성/갱신되는 데이터 파일입니다 (git에는 포함되지 않음). |

## 실행 방법

```
python app.py
```

실행하면 아래와 같은 메뉴가 반복해서 나타납니다. 숫자를 입력해 원하는 기능을 선택하세요.

```
===============================
1. Create
2. Read (전체 목록)
3. Read (ID/이름 검색)
4. Update
5. Delete
0. 종료
===============================
선택 >
```

### 메뉴별 사용법

- **1. Create** — 이름과 값을 입력받아 새 데이터를 만들고 `data.json`에 저장합니다. ID는 자동으로 1씩 증가하며 부여됩니다.
- **2. Read (전체 목록)** — 저장된 데이터를 전부 보여줍니다.
- **3. Read (ID/이름 검색)** — ID 또는 이름으로 특정 데이터를 검색합니다.
- **4. Update** — 수정할 데이터의 ID를 입력한 뒤, 어떤 필드(`name` 또는 `value`)를 어떤 값으로 바꿀지 입력합니다.
- **5. Delete** — 삭제할 데이터의 ID를 입력하면, 실수로 지우는 것을 막기 위해 한 번 더 확인(`y`/`N`)을 받은 후 삭제합니다.
- **0. 종료** — 프로그램을 종료합니다.

### 사용 예시

```
선택 > 1
이름: apple
값: 100
생성됨: {'id': 1, 'name': 'apple', 'value': '100'}

선택 > 2
{'id': 1, 'name': 'apple', 'value': '100'}

선택 > 4
수정할 ID: 1
수정할 필드 (name, value): value
새 값: 999
수정됨: {'id': 1, 'name': 'apple', 'value': '999'}

선택 > 5
삭제할 ID: 1
{'id': 1, 'name': 'apple', 'value': '999'} 를 정말 삭제할까요? (y/N): y
삭제되었습니다.
```

데이터는 아래와 같이 `data.json`에 사람이 읽기 쉬운 형태로 저장됩니다.

```json
[
  {
    "id": 1,
    "name": "apple",
    "value": "999"
  }
]
```

## 테스트 실행

```
pip install pytest
pytest tests
```

`tests/test_app.py`는 실제 `data.json`을 건드리지 않도록 임시 파일을 사용하며, 입력(`input()`)도 가짜 값으로 대체해서 검증합니다.
